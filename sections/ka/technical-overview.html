1 პლატფორმის მიმოხილვა

MediMind EMR აგებულია თანამედროვე, სტანდარტიზებულ ტექნოლოგიებზე, რომელიც უზრუნველყოფს სისტემათა თავსებადობას, მასშტაბურობას და უსაფრთხოებას.

რატომ FHIR?

FHIR (Fast Healthcare Interoperability Resources) არის HL7-ის მიერ შემუშავებული თანამედროვე სტანდარტი ჯანდაცვის მონაცემების გაცვლისთვის.

FHIR R4 სტანდარტი: MediMind იყენებს FHIR R4 (4.0.1) სტანდარტს - ყველაზე ფართოდ გავრცელებულსა და სტაბილურს.

FHIR-ის უპირატესობები:

  • სისტემათა თავსებადობა - სტანდარტიზებული API საშუალებას იძლევა სხვადასხვა სისტემებთან ინტეგრაციისთვის
  • RESTful არქიტექტურა - მარტივი HTTP მეთოდები (GET, POST, PUT, DELETE) რესურსებთან სამუშაოდ
  • JSON/XML ფორმატები - თანამედროვე, ადვილად წაკითხვადი მონაცემთა ფორმატები
  • მოდულარობა - 150+ რესურსის ტიპი, საჭიროებისამებრ გამოყენება
  • გლობალური მხარდაჭერა - მსოფლიოს მასშტაბით აღიარებული სტანდარტი

React კომპონენტები

Medplum-ის ძირითადი კომპონენტები:

  • FHIR API - RESTful API მონაცემების მიღება/გაგზავნისთვის
  • TypeScript SDK - კლიენტ-ბიბლიოთეკა API-სთან მუშაობისთვის
  • React Components - მზა UI კომპონენტები ჯანდაცვის აპლიკაციებისთვის
  • Medplum Bots - სერვერული ლოგიკის გაშვება
  • Admin Console - მონაცემების მართვის ვებ-ინტერფეისი
