2 Features

2.1 Patient Search System

The registration module features a powerful UnifiedPatientSearch component that serves as the primary way to find existing patients.

Hero Search Component

The hero search bar is prominently displayed at the top of the registration page:

Feature Description Keyboard Shortcut
Quick Search Auto-detects if input is personal ID (11 digits) or name Cmd/Ctrl + K
Advanced Filters Expand panel for Personal ID, First Name, Last Name, Reg Number Cmd/Ctrl + Shift + K
Navigate Results Move through search results ↑ / ↓
Select Result Choose highlighted result Enter
Close Search Close search panel Escape

Advanced Filters Panel

Click the "Advanced" button or use Cmd/Ctrl + Shift + K to open detailed filters:

  • Personal ID - 11-digit Georgian personal number
  • First Name - Patient's given name
  • Last Name - Patient's family name
  • Registration Number - Internal EMR registration number

Search Results with Dual Actions

When you search for a patient by name or personal ID, matching results appear in a dropdown with two action buttons:

Each search result displays two action buttons:

New Visit
Creates a new visit for this patient and navigates to visit details
Fill Form
Populates the registration form with patient data for editing

Patient Lookup Section

Within the form, the PatientLookupSection provides a dedicated field for looking up patients by their 11-digit personal ID:

How Patient Lookup Works:
  • Enter an 11-digit personal ID
  • System searches with 300ms debounce
  • If found: displays patient card with name, DOB, visit count
  • If not found: allows new patient registration
  • When patient found: form sections auto-collapse

Patient Found Card

When an existing patient is found, a card displays their details with options to edit or create a new visit:

Create Visit Toggle

The form includes a toggle to control whether a visit is created along with the patient:

Create Visit: ON

Patient + Visit created together. Registration, Insurance, and Guarantee sections visible.

Create Visit: OFF

Patient only created. Visit-related sections hidden.

The following diagram illustrates the complete patient registration workflow:

Draft Auto-Save

The form automatically saves your progress to localStorage:

Feature Behavior
Auto-Save Interval 1-second debounce after any field change
Draft Indicator Shows "Draft saved" badge when unsaved changes exist
Clear Draft Button to discard saved draft and reset form
Storage Key medimind-registration-draft

2.3 Document Upload

The Guarantee section includes a document upload feature for attaching guarantee letters and supporting documents:

Upload Features

  • Drag & Drop - Drag files directly onto the upload zone
  • Click to Browse - Click the zone to open file picker
  • Multiple Files - Up to 5 files per guarantee
  • Multiple Guarantees - Up to 3 guarantees per registration
  • Document Metadata - Name, date, author, document number

Document Fields

Field Description Required
Document Name Descriptive name for the document Yes
Document Date Date the document was issued Yes
Author Person or organization that issued the document No
Document Number Reference number on the document No
File Attachment The actual file (PDF, image, etc.) Yes
FHIR Resources Created:
  • Binary - Stores the actual file content
  • DocumentReference - Contains metadata and links to Binary

Planned

Recent Patients

Shows the 10 most recently updated EMR patients (excludes portal patients):

  • Patient Name - Full name
  • Personal ID - 11-digit number
  • Date of Birth - Birth date
  • Actions:
    • New Visit - Create a new visit for this patient
    • View - Open patient's full record

2.6 Visit Management

When the createVisit toggle is enabled, visit-related sections become available in the form:

Creating a New Encounter

After finding an existing patient, the form shows their details and the visit registration fields for creating a new encounter:

Registration Section

Field Description
Visit Type Outpatient, Inpatient, or Emergency
Visit Date Date of the visit
Department Target department/service
Referring Doctor Doctor who referred the patient (optional)

Insurance Section

Add up to 3 insurance companies per visit:

  • Insurance Company - Select from registered insurers
  • Policy Number - Patient's policy number
  • Coverage Type - Full, partial, or specific coverage
  • Add/Remove Buttons - Manage multiple insurers

Active Visit Warning

If a patient already has an in-progress visit, a warning modal appears:

Active Visit Detection:

The system prevents duplicate encounters by detecting in-progress visits. You can choose to:

  • View the existing active visit
  • Create a new visit anyway (with confirmation)
  • Cancel and return to the form

Demographics Section

Additional demographic data collected during visit registration:

  • Region - Patient's residential region
  • Chief Complaint - Primary reason for visit
  • Referral Source - How patient was referred
