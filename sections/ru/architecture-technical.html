Техническое описание

Модуль регистрации построен на Layer-based архитектуре, где каждый уровень отвечает за конкретный функционал и коммуникация происходит только между соседними уровнями.

Технологический стек:
  • Frontend: React 18 + TypeScript 5.x
  • Backend: Medplum FHIR R4 Server (api.medplum.com)
  • State Management: React Hooks + Context API
  • UI Framework: Mantine UI Components
Уровень Ответственность Примеры
UI Layer Пользовательский интерфейс, формы, обработка событий RegistrationVisitModal, PatientForm, RegistrationVisitWizard
Hooks Layer Бизнес-логика, state management, side effects useRegistrationVisitForm, useDepartments
Services Layer Коммуникация с FHIR API, трансформация данных encounterService, departmentService
FHIR API Персистенция данных, валидация Medplum Cloud (Patient, Encounter, Coverage)

Система валидации

Валидация формы осуществляется модулем validators в реальном времени.

Поле Правило Сообщение об ошибке
Личный номер Ровно 11 цифр (/^\d{11}$/) "Личный номер должен содержать 11 цифр"
Эл. почта Формат RFC 5322 "Неверный формат эл. почты"
Дата рождения Не в будущем, не более 120 лет "Дата не может быть в будущем"
Телефон Международный формат (+995...) "Неверный формат телефона"

Обнаружение дубликатов

При вводе личного номера система автоматически проверяет существует ли такой пациент:

  • Если найден — появляется окно сравнения
  • Можно выбрать существующего или создать нового
  • Предотвращаются дублирующиеся записи

Поток данных (Data Flow)

Движение данных происходит однонаправленно: 

UI Component → useRegistrationVisitForm hook → Service → Medplum FHIR API
Процесс Уровень Описание
Заполнение формы UI → Hook onChange → setFormData state update
Сохранение Draft Hook → localStorage 1-секундный debounce, JSON сериализация
Сохранение пациента Hook → Service → API medplum.createResource('Patient', data)
Создание визита Hook → Service → API encounterService.createEncounter()
Recent Patients Cache: Последние 10 посещённых пациентов сохраняются в localStorage с помощью recentPatientsService (FIFO - First In First Out).

Система использует стандарт FHIR R4. Основные ресурсы:

Ресурс Patient

Поле FHIR Path Описание
Личный номер identifier[system="NATIONAL_ID"] 11-значный уникальный идентификатор
Имя name[0].given[0] Имя пациента
Фамилия name[0].family Фамилия пациента

Пол gender male | female | other | unknown

Поле FHIR Path Описание

Поле FHIR Path Описание

Поле FHIR Path Описание

Регистрация неизвестного пациента

Для экстренных случаев, когда пациент без сознания или невозможно установить личность.

Автоматически генерируется:

  • Временный ID: в формате UNK-2026-001
  • Системное имя: «Неизвестный #1», «Неизвестный #2»...
  • Время прибытия: текущая дата и время

Важно: Данные неизвестного пациента должны быть обновлены сразу после установления личности. Система показывает напоминание о таких пациентах.
