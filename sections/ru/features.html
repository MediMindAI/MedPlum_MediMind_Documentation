2 Функции

2.1 Система поиска пациентов

Модуль регистрации включает мощный компонент UnifiedPatientSearch, который служит основным способом поиска существующих пациентов.

Компонент главного поиска

Строка главного поиска отображается вверху страницы регистрации:

Функция Описание Сочетание клавиш
Быстрый поиск Автоопределение: личный номер (11 цифр) или имя Cmd/Ctrl + K
Расширенные фильтры Панель: ИИН, Имя, Фамилия, Рег. номер Cmd/Ctrl + Shift + K
Навигация по результатам Перемещение по результатам поиска ↑ / ↓
Выбор результата Выбор выделенного результата Enter
Закрыть поиск Закрыть панель поиска Escape

Панель расширенных фильтров

Нажмите кнопку «Расширенный» или используйте Cmd/Ctrl + Shift + K для открытия детальных фильтров:

  • Личный номер — 11-значный грузинский личный номер
  • Имя — Имя пациента
  • Фамилия — Фамилия пациента
  • Регистрационный номер — Внутренний регистрационный номер EMR

Результаты поиска с двойными действиями

При поиске пациента по имени или личному номеру результаты отображаются в выпадающем списке с двумя кнопками действий:

Каждый результат поиска отображает две кнопки действий:

Новый визит
Создает новый визит для пациента и переходит к деталям визита
Заполнить форму
Заполняет форму регистрации данными пациента для редактирования

Секция поиска по ИИН

В форме компонент PatientLookupSection предоставляет поле для поиска пациентов по 11-значному личному номеру:

Как работает поиск по ИИН:
  • Введите 11-значный личный номер
  • Система ищет с задержкой 300мс
  • Если найден: отображается карточка пациента с именем, датой рождения, количеством визитов
  • Если не найден: разрешается регистрация нового пациента
  • При нахождении пациента: секции формы автоматически сворачиваются

Карточка найденного пациента

При нахождении существующего пациента карточка отображает его данные с возможностью редактирования или создания нового визита:

2.2 Единая форма регистрации

Форма регистрации организована в 8 сворачиваемых секций. Каждую секцию можно развернуть или свернуть для фокусировки на нужных полях.

1
Поиск пациента
Поиск по 11-значному ИИН с карточкой
2
Личная информация
Имя, пол, дата рождения, переключатель визита
3
Контактная информация
Телефон, email, адрес
4
Дополнительные данные
Гражданство, семейное положение, работа
5
Опекун/Представитель
Обязательно для пациентов до 18 лет
6
Регистрация
Детали визита (опционально)
7
Страхование
1-3 страховщика с добавлением/удалением
8
Гарантия
Письма с загрузкой документов

Переключатель создания визита

Форма включает переключатель для управления созданием визита вместе с пациентом:

Создать визит: ВКЛ

Пациент + Визит создаются вместе. Секции Регистрация, Страхование и Гарантия видимы.

Создать визит: ВЫКЛ

Создается только пациент. Секции, связанные с визитом, скрыты.

Поток регистрации

Следующая диаграмма иллюстрирует полный рабочий процесс регистрации пациента:

flowchart TD A[Patient Lookup] --> B{Patient Found?} B -->|Yes| C[Auto-fill Form] B -->|No| D[New Patient Entry] C --> E{Create Visit?} D --> E E -->|Yes| F[Fill Visit Details] E -->|No| G[Patient Only] F --> H[Submit] G --> H H --> I[FHIR Resources Created]

Автосохранение черновика

Форма автоматически сохраняет прогресс в localStorage:

Функция Поведение
Интервал автосохранения Задержка 1 секунда после любого изменения поля
Индикатор черновика Показывает бейдж «Черновик сохранен» при наличии несохраненных изменений
Очистить черновик Кнопка для удаления сохраненного черновика и сброса формы
Ключ хранения medimind-registration-draft

2.3 Загрузка документов

Секция Гарантия включает функцию загрузки документов для прикрепления гарантийных писем и сопроводительных документов:

Функции загрузки

  • Drag & Drop — Перетаскивайте файлы прямо в зону загрузки
  • Клик для выбора — Нажмите на зону для открытия диалога выбора файлов
  • Множество файлов — До 5 файлов на одну гарантию
  • Множество гарантий — До 3 гарантий на регистрацию
  • Метаданные документа — Название, дата, автор, номер документа

Поля документа

Поле Описание Обязательно
Название документа Описательное название документа Да
Дата документа Дата выдачи документа Да
Автор Лицо или организация, выдавшая документ Нет
Номер документа Регистрационный номер на документе Нет
Файл вложения Сам файл (PDF, изображение и т.д.) Да
Создаваемые FHIR-ресурсы:
  • Binary — Хранит содержимое файла
  • DocumentReference — Содержит метаданные и ссылку на Binary

2.5 Мобильный мастер

На экранах уже 768px форма трансформируется в 5-шаговый мастер для удобной мобильной навигации:

1
Пациент
Поиск + Личная информация
2
Детали
Контакты + Дополнительно + Опекун
3
Визит
Детали регистрации (если createVisit=true)
4
Страхование
Страхование + Гарантия (если createVisit=true)
5
Подтверждение
Демография + Сводка

Поток шагов мастера

flowchart LR S1[Step 1: Patient] --> S2[Step 2: Details] S2 --> S3[Step 3: Visit] S3 --> S4[Step 4: Insurance] S4 --> S5[Step 5: Confirm]
Поведение мобильного мастера:
  • Шаги 3 и 4 скрыты когда переключатель createVisit ВЫКЛ
  • Индикатор прогресса показывает текущий шаг
  • Кнопки Назад/Далее для навигации между шагами
  • Автосохранение черновика работает на всех шагах

2.6 Управление визитами

Когда переключатель createVisit включен, секции, связанные с визитом, становятся доступными в форме:

Создание нового визита

После нахождения существующего пациента форма отображает его данные и поля регистрации визита для создания нового визита:

Секция регистрации

Поле Описание
Тип визита Амбулаторный, Стационарный или Экстренный
Дата визита Дата визита
Отделение Целевое отделение/служба
Направляющий врач Врач, направивший пациента (опционально)

Секция страхования

Добавьте до 3 страховых компаний на один визит:

  • Страховая компания — Выбор из зарегистрированных страховщиков
  • Номер полиса — Номер полиса пациента
  • Тип покрытия — Полное, частичное или специфическое покрытие
  • Кнопки Добавить/Удалить — Управление несколькими страховщиками

Предупреждение об активном визите

Если у пациента уже есть визит в процессе, появляется предупреждающее окно:

Обнаружение активного визита:

Система предотвращает дублирование записей, обнаруживая визиты в процессе. Вы можете:

  • Просмотреть существующий активный визит
  • Создать новый визит в любом случае (с подтверждением)
  • Отменить и вернуться к форме

Секция демографии

Дополнительные демографические данные, собираемые при регистрации визита:

  • Регион — Регион проживания пациента
  • Основная жалоба — Основная причина визита
  • Источник направления — Как пациент был направлен
